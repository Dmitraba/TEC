Decision Frame

Что выбрать вместо Telegram-бота

Onsite-чат Для быстрых вопросов прямо на сайте
AI-консультант Для ответов по PDF, моделям, базе знаний и FAQ
Подборщик Для сбора параметров и передачи лида в форму
Гибрид Чат + форма подбора как основной B2B-сценарий
Step by Step

Порядок создания

1. Определи роль

Это FAQ-бот, AI-консультант, подборщик, гибрид или лид-форма с логикой?

2. Определи границы

На что бот может отвечать, а где должен честно сказать: данных нет.

3. Собери источники

PDF, карточки моделей, FAQ, контакты, формы, сценарии CTA.

4. Настрой ответы

Короткие, прямые, без фантазий, с маршрутом в консультацию или КП.

5. Настрой fallback

Если данных нет, бот не сочиняет, а переводит в форму или к человеку.

6. Проверь канал

Telegram только как дополнительный маршрут, не как единственный.

Telegram может использоваться как дополнительный канал, но критическую коммерческую воронку надежнее держать внутри сайта или в контролируемом продукте.
Prompt Sample

Образец prompt для создания бота

Create a practical customer-facing assistant for a technical product website.

Goals:
- answer user questions clearly
- rely only on confirmed source materials
- help users request a quote, specification, or consultation
- reduce hallucinations
- escalate to a human or form when needed

Rules:
- do not invent product facts
- if the answer is not present in the source materials, say that clearly
- use short, direct, professional wording
- keep terminology consistent with the source documents
- treat Telegram as an optional secondary channel, not the only critical channel

Capabilities:
- answer FAQ
- explain product models and differences
- help choose the right model
- offer CTA paths:
  - request quote
  - request specification
  - get consultation
  - submit selection request
Checklist

Что проверить перед запуском

  • Бот опирается на подтвержденные материалы, а не на вольные формулировки.
  • Есть сценарий, когда бот не знает ответ и честно это говорит.
  • Есть понятный переход в заявку, КП, спецификацию или менеджеру.
  • Telegram не является единственной критической точкой контакта.
  • Терминология совпадает с продуктовой документацией.
